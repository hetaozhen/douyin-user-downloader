# 抖音下载器开发与迭代规则

当该项目的功能发生版本进化、重构或行为变更时，开发助手必须遵守以下规范：

1. **版本更新后更新文档 (MD)**：
   - 必须及时更新项目中的 `README.md` 和 `README_EN.md`，将新增或修改的功能特性（例如：单视频极速下载、智能文本链接提取、发布时间/关键字过滤、文件名格式优化等）同步记录并丰富到文档中。

2. **Git 版本控制记录**：
   - 在开发和验证完成后，主动或提醒用户运行 `git add .` 和 `git commit -m` 提交版本，完整保留每一次迭代进化的提交记录（例如 `feat: ...`, `fix: ...`）。

3. **终端编码兼容**：
   - 如果需要在 Windows 控制台（CMD/Powershell）中打印带有 Emoji 或特殊 Unicode 字符的日志，必须确保在 `main()` 启动时通过 `sys.stdout.reconfigure(encoding='utf-8')` 配置正确的编码，防止抛出 `UnicodeEncodeError`。
